# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

FluSi — a simple flight simulator built as a gift for the author's six-year-old son. Runs on macOS and Linux.

## Tech Stack

- Unity **6000.3.19f1** (Unity 6) with C#, Universal Render Pipeline (URP).
- New Input System (`com.unity.inputsystem`) — input is defined in `Assets/InputSystem_Actions.inputactions`, not the legacy Input Manager.
- Test Framework (`com.unity.test-framework`) available for EditMode/PlayMode tests.

## Setup

```shell
brew install --cask unity-hub
brew install --cask dotnet-sdk
```

Open the project in Unity Hub, which will install editor `6000.3.19f1` if missing.

## Commands

`Unity` is the editor binary (macOS: `/Applications/Unity/Hub/Editor/6000.3.19f1/Unity.app/Contents/MacOS/Unity`). Close the Editor before running batchmode commands — Unity locks the project.

```shell
# Run all tests (EditMode or PlayMode) and exit
Unity -runTests -batchmode -projectPath . -testPlatform EditMode -testResults results.xml
Unity -runTests -batchmode -projectPath . -testPlatform PlayMode -testResults results.xml

# Filter to a single test / fixture
Unity -runTests -batchmode -projectPath . -testPlatform EditMode -testFilter "Namespace.Fixture.TestName" -testResults results.xml
```

C# is compiled by the Editor, not `dotnet`. The `.csproj`/`.sln` files are generated by Unity (and gitignored) purely for IDE/LSP support — do not hand-edit them; regenerate via the Editor if stale.

## Architecture Notes

- This is currently the stock Unity URP template. The only C# (`Assets/TutorialInfo/`) is Unity's sample `Readme` viewer and can be deleted once real gameplay begins.
- Game logic lives in `Assets/` — scenes in `Assets/Scenes/` (`SampleScene.unity` is the entry scene), render/quality config in `Assets/Settings/` (separate `PC_*` and `Mobile_*` renderer/RP assets).
- Every asset has a paired `.meta` file holding its GUID; references between assets are by GUID. Always let Unity move/rename/delete assets (or move the `.meta` alongside) so references don't break, and commit `.meta` files.
- `Library/`, `Temp/`, `obj/`, `Logs/`, `UserSettings/`, and build output are gitignored regenerable caches — never edit or rely on them.
